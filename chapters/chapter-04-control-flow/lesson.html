Control Flow: if / elif / else

Every interesting program eventually needs to make a decision. This is the chapter where your code starts to think.

On this page 1. Simple if 2. if-else 3. elif chains 4. Nested if
So far, every program you've written runs the exact same lines every single time, top to bottom. Real software doesn't work that way — a login check either lets you in or doesn't, a discount either applies or doesn't, a grade is A or B or C depending on the score. if is how your program branches based on what's actually true right now.
Sub-topic 1 of 4

1. Simple if

An if statement checks a condition. If it's True, the indented block underneath runs. If it's False, Python skips that block entirely and moves on.

simple_if.py 
age = 20

if age >= 18:
    print("You can vote.")

print("Program continues here either way.")
try_if.py
Click "Run Code" — try changing 16 to 20 and running again.
What is... indentation? In most languages, curly braces { } mark a block of code. Python uses indentation (consistent spacing, usually 4 spaces) instead — every line indented under the if is part of that block. This isn't just a style preference; it's how Python knows where the block ends. Mixing tabs and spaces, or indenting inconsistently, causes a real error.
Go Deeper: why colon + indentation instead of braces?

Python's designer, Guido van Rossum, wanted code layout to be forced to match code structure — in languages with braces, you can indent however you like and the compiler doesn't care, which sometimes lets sloppy or misleading formatting slip through. By making indentation the actual syntax, Python code you read is guaranteed to be structured the way it looks.

Quick recap
  • if condition: then an indented block runs only when the condition is True.
  • Indentation is not decoration — it's how Python defines the block.
  • Code after the if block, back at the original indent level, always runs.
Sub-topic 2 of 4

2. if-else

Often you want to do one thing OR another — never both, never neither. else catches every case the if didn't.

if_else.py 
number = 7

if number % 2 == 0:
    print(number, "is even")
else:
    print(number, "is odd")
What is... a boolean expression? A boolean expression is anything that evaluates to True or False — usually built from the relational and logical operators from Chapter 3. number % 2 == 0 is a boolean expression: it's either True (even) or False (odd), never anything in between.
Quick recap
  • else: runs exactly when the if condition was False.
  • Exactly one of the two blocks runs — never both, never neither.
Sub-topic 3 of 4

3. elif — checking more than two possibilities

Real decisions often have more than two outcomes. elif (short for "else if") lets you check condition after condition, in order, stopping at the first one that's true.

grade.py 
average = 68

if average >= 75:
    grade = "A"
elif average >= 60:
    grade = "B"
elif average >= 50:
    grade = "C"
elif average >= 40:
    grade = "D"
else:
    grade = "Fail"

print("Grade:", grade)   # Grade: B
try_grade.py
Click "Run Code" — try a few different average values.
Go Deeper: order matters in an elif chain

Python checks each condition top to bottom and stops at the very first True one — it never checks the rest, even if they'd also be true. This is why the grade example checks >= 75 before >= 60: if it checked >= 40 first, a score of 90 would incorrectly match that condition first and stop there, before ever reaching >= 75. Always order an elif chain from most specific (or highest threshold) to least.

Quick recap
  • elif checks another condition only if the previous ones were False.
  • Only the first matching branch runs — order your conditions carefully.
  • else at the end catches everything that didn't match any condition.
Sub-topic 4 of 4

4. Nested if — decisions inside decisions

An if can contain another if inside its block. This is useful when the second question only makes sense after the first is already answered.

nested_if.py 
a = 10
b = 5
c = 4

if a > b:
    if a > c:
        print("a is the biggest:", a)
Go Deeper: nested if vs. a single "and"

The example above could also be written as one line: if a > b and a > c:. Both are correct here. Nesting becomes genuinely necessary once the inner decision needs its own separate else, or depends on something only computed inside the outer block — you'll see real examples of this once you reach loops and functions.

Quick recap
  • An if inside another if's block is called nested.
  • The inner condition is only checked if the outer one was already True.

Points to Remember

  • if condition: runs an indented block only when the condition is true.
  • Indentation defines the block in Python — there are no curly braces.
  • else catches everything the if didn't; exactly one branch runs.
  • elif chains check multiple possibilities in order, stopping at the first match — order high-to-low or most-specific-first.
  • Nested if statements handle decisions that only make sense after an outer condition is already true.
