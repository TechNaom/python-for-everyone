2 Safe Index Sum Easy

Write a function safe_index_sum(values, indices) that sums values[i] for every i in indices, but skips any index that's out of range instead of raising IndexError.

Constraints: values is a list of numbers; indices is a list of ints (may include negative or too-large indices).
Examples:
safe_index_sum([1, 2, 3], [0, 1, 2]) → 6
safe_index_sum([1, 2, 3], [0, 5, 10]) → 1
challenge_2.py
Show Solution
solution_2.py 
def safe_index_sum(values, indices):
    total = 0
    for i in indices:
        if 0 <= i < len(values):
            total += values[i]
    return total

8 Will the Countdown Terminate? Medium

A recursive countdown(n) function repeatedly adds step to n until n reaches exactly 0, then stops. Write a function will_terminate(n, step) that predicts — without actually recursing — whether that countdown will ever land exactly on 0 (True) or would recurse forever / hit RecursionError (False).

Constraints: n and step are integers. If step is 0, the countdown never changes, so it terminates only if n is already 0.
Examples:
will_terminate(5, -1) → True (5, 4, 3, 2, 1, 0)
will_terminate(-5, -1) → False (-5, -6, -7, ... never reaches 0)
will_terminate(5, -2) → False (5, 3, 1, -1, -3, ... skips over 0)
challenge_8.py
Show Solution
solution_8.py 
def will_terminate(n, step):
    if step == 0:
        return n == 0
    if n == 0:
        return True
    # moving away from 0 -- n positive and step non-negative, or
    # n negative and step non-positive -- can never reach 0
    if (n > 0) != (step < 0):
        return False
    # otherwise, n reaches exactly 0 only if it's an exact multiple
    # of step in the direction that closes the distance to 0
    return n % step == 0

9 Has a Duplicate? Easy

Write a function has_duplicate(items) that returns True if any value appears more than once in items, or False otherwise. Use a set for the membership check, not repeated in checks against a list.

Constraints: items is a list of hashable values (numbers or strings).
Examples:
has_duplicate([1, 2, 3, 2]) → True
has_duplicate([1, 2, 3]) → False

Show Solution
solution_9.py 
def has_duplicate(items):
    seen = set()
    for item in items:
        if item in seen:
            return True
        seen.add(item)
    return False

11 Cached Uppercase Medium

Write a function cached_upper(keys) that returns a list [results, call_count], where results is key.upper() for every key in keys (in order), and call_count is how many times .upper() actually had to be computed — cache repeated keys so a repeated key is never recomputed.

Constraints: keys is a list of strings.
Examples:
cached_upper(["a", "b", "a"]) → [["A", "B", "A"], 2] (2 distinct keys, so only 2 real computations even though 3 lookups happened)
challenge_11.py
Show Solution
solution_11.py 
def cached_upper(keys):
    cache = {}
    call_count = 0
    results = []
    for key in keys:
        if key not in cache:
            call_count += 1
            cache[key] = key.upper()
        results.append(cache[key])
    return [results, call_count]

    for n in numbers:
        doubled = n * 2
        if doubled > threshold:
            return doubled
    return None

16 Bounded Cache Simulator Medium

Write a function bounded_cache_final_state(keys, maxsize) that simulates a simple least-recently-inserted cache: process keys in order, inserting each into the cache (a key already present is not re-inserted or moved). Once the cache would exceed maxsize entries, evict the oldest-inserted key first. Return the final list of keys still in the cache, oldest-inserted first.

Constraints: keys is a list of hashable values; maxsize is a positive int.
Examples:
bounded_cache_final_state(["a", "b", "c"], 2) → ["b", "c"] (a is evicted once c pushes past size 2)
bounded_cache_final_state(["a", "a", "b"], 2) → ["a", "b"] (repeated "a" is not re-inserted)
challenge_16.py
Show Solution
solution_16.py 
def bounded_cache_final_state(keys, maxsize):
    cache = []
    for key in keys:
        if key in cache:
            continue
        cache.append(key)
        if len(cache) > maxsize:
            cache.pop(0)
    return cache

Write a function process_charge(processed, idempotency_key, amount) that simulates a safe retry: processed is a dict mapping idempotency keys already charged to their amount. If idempotency_key is already in processed, return its existing recorded amount unchanged (don't charge again). Otherwise, record amount under that key in processed and return it.

Constraints: processed is a dict (mutated in place); idempotency_key is a string; amount is a number.
Examples:
First call process_charge({}, "req-1", 50) → 50, and processed now has {"req-1": 50}.
A retried call process_charge({"req-1": 50}, "req-1", 50) → 50 (not charged twice).
challenge_27.py
Show Solution
solution_27.py 
def process_charge(processed, idempotency_key, amount):
    if idempotency_key in processed:
        return processed[idempotency_key]
    processed[idempotency_key] = amount
    return amount

33 Symptom Classifier Hard

Write a function classify(description, catalog) where catalog is a list of dicts each with "id" and "keywords" (a list of lowercase strings). Return a sorted list of every catalog entry's "id" whose keywords list has at least one keyword that's a substring of description, checked case-insensitively — each matching id should appear exactly once, even if multiple of its keywords match.

Constraints: description is a string (any case); catalog is a list of dicts as described.
Examples:
classify("Getting a KEYERROR in prod", [{"id": 1, "keywords": ["keyerror"]}, {"id": 2, "keywords": ["timeout"]}]) → [1]
challenge_33.py
Show Solution
solution_33.py 
def classify(description, catalog):
    lowered = description.lower()
    matched_ids = [
        entry["id"] for entry in catalog
        if any(keyword in lowered for keyword in entry["keywords"])
    ]
    return sorted(matched_ids)
