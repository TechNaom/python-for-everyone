2 Safe Index Sum Easy

Write a function safe_index_sum(values, indices) that sums values[i] for every i in indices, but skips any index that's out of range instead of raising IndexError.

Constraints: values is a list of numbers; indices is a list of ints (may include negative or too-large indices).
Examples:
safe_index_sum([1, 2, 3], [0, 1, 2]) → 6
safe_index_sum([1, 2, 3], [0, 5, 10]) → 1
challenge_2.py
Show Solution
solution_2.py 
def safe_index_sum(values, indices):
    total = 0
    for i in indices:
        if 0 <= i < len(values):
            total += values[i]
    return total

def find_index_or_negative_one(items, target):
    for i, item in enumerate(items):
        if item == target:
            return i
    return -1

        return False

8 Will the Countdown Terminate? Medium

A recursive countdown(n) function repeatedly adds step to n until n reaches exactly 0, then stops. Write a function will_terminate(n, step) that predicts — without actually recursing — whether that countdown will ever land exactly on 0 (True) or would recurse forever / hit RecursionError (False).

Constraints: n and step are integers. If step is 0, the countdown never changes, so it terminates only if n is already 0.
Examples:
will_terminate(5, -1) → True (5, 4, 3, 2, 1, 0)
will_terminate(-5, -1) → False (-5, -6, -7, ... never reaches 0)
will_terminate(5, -2) → False (5, 3, 1, -1, -3, ... skips over 0)
challenge_8.py
Show Solution
solution_8.py 
def will_terminate(n, step):
    if step == 0:
        return n == 0
    if n == 0:
        return True
    # moving away from 0 -- n positive and step non-negative, or
    # n negative and step non-positive -- can never reach 0
    if (n > 0) != (step < 0):
        return False
    # otherwise, n reaches exactly 0 only if it's an exact multiple
    # of step in the direction that closes the distance to 0
    return n % step == 0

9 Has a Duplicate? Easy

Write a function has_duplicate(items) that returns True if any value appears more than once in items, or False otherwise. Use a set for the membership check, not repeated in checks against a list.

Constraints: items is a list of hashable values (numbers or strings).
Examples:
has_duplicate([1, 2, 3, 2]) → True
has_duplicate([1, 2, 3]) → False
challenge_9.py
Show Solution
solution_9.py 
def has_duplicate(items):
    seen = set()
    for item in items:
        if item in seen:
            return True
        seen.add(item)
    return False

10 Reverse Without Inserting at the Front Easy

Write a function reverse_via_append(items) that returns a new list with items in reverse order, using only .append() and .reverse() — never .insert(0, x).

Constraints: items is a list.
Examples:
reverse_via_append([1, 2, 3]) → [3, 2, 1]

solution_10.py 
def reverse_via_append(items):
    result = []
    for item in items:
        result.append(item)
    result.reverse()
    return result

11 Cached Uppercase Medium

Write a function cached_upper(keys) that returns a list [results, call_count], where results is key.upper() for every key in keys (in order), and call_count is how many times .upper() actually had to be computed — cache repeated keys so a repeated key is never recomputed.

Constraints: keys is a list of strings.
Examples:
cached_upper(["a", "b", "a"]) → [["A", "B", "A"], 2] (2 distinct keys, so only 2 real computations even though 3 lookups happened)
challenge_11.py
Show Solution
solution_11.py 
def cached_upper(keys):
    cache = {}
    call_count = 0
    results = []
    for key in keys:
        if key not in cache:
            call_count += 1
            cache[key] = key.upper()
        results.append(cache[key])
    return [results, call_count]

12 Index Records by ID Medium

Write a function index_by_id(records) that builds a dict mapping each record's "id" value to the record itself, then returns it as a list of [id, record] pairs, sorted by id — the one-time setup step that turns a slow nested-loop lookup into a fast dict lookup.

Constraints: records is a list of dicts, each with an "id" key.
Examples:
index_by_id([{"id": 1, "name": "x"}]) → [[1, {"id": 1, "name": "x"}]]
challenge_12.py
Show Solution
solution_12.py 
def index_by_id(records):
    by_id = {r["id"]: r for r in records}
    return [[k, v] for k, v in sorted(by_id.items())]

14 Stop Early Medium

Write a function first_over_threshold(numbers, threshold) that returns the first value in numbers, doubled, that's greater than threshold — or None if none qualify. Loop and return as soon as a match is found; don't build a full list of every doubled value first.

Constraints: numbers is a list of numbers; threshold is a number.
Examples:
first_over_threshold([1, 2, 3, 4, 5], 3) → 4 (2 doubled is 4, the first doubled value over 3)
challenge_14.py
Show Solution
solution_14.py 
def first_over_threshold(numbers, threshold):
    for n in numbers:
        doubled = n * 2
        if doubled > threshold:
            return doubled
    return None
