Exercises: Debugging Production Issues

How to run these Download starter.py below, then run python3 starter.py from inside the folder you saved it in. Every task uses only the standard library — no installs needed.
starter.py solution.py (reference)

1 A safe dict lookup

Write safe_lookup(config, key, default) — return the value if present, otherwise default, never raising KeyError.

2 A safe top-N sum

Write safe_top_n(scores, n) — sum the first n items without ever indexing past the end of the list.

3 Catching the specific exception

Write safe_int_lookup(raw, key, default) — catch only KeyError, not a bare except:.

4 Validating input before arithmetic

Write safe_discount(price, percent_off) — convert percent_off to a float before using it.

5 Raising instead of implicitly returning None

Write find_user_or_raise(users, name) — raise a clear ValueError instead of implicitly returning None.

6 Handling a zero denominator

Write average_or_none(total, count) — return None instead of crashing when count is 0.

7 Handling a non-JSON response

Write parse_json_safe(raw_text) — catch json.JSONDecodeError specifically and return None.

🐞 Task 8 — Debug the Code

This is supposed to return 30 if the "timeout" key is missing, but its bare except: also hides a real typo in the config dict below ("timeoout", not "timeout"). Fix the except clause so it only catches KeyError.

buggy_load_timeout.py 
def load_timeout(raw):
    try:
        return int(raw["timeout"])
    except:
        return 30


raw_config = {"timeoout": "45"}
print("load_timeout result:", load_timeout(raw_config))
