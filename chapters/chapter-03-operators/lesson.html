Operators

You've used + already. Now let's meet the rest of the toolbox that lets your programs actually compute things.

On this page 1. Arithmetic & assignment 2. Relational & logical 3. Bitwise 4. Special: is / in
An operator is a tiny symbol that does real work: + adds, > compares, and combines conditions. Every piece of logic you'll ever write — a discount calculation, a login check, a permission system — is built entirely out of these small pieces stacked together.
Sub-topic 1 of 4

1. Arithmetic & assignment operators

Python has two division operators that behave differently — this single detail causes more beginner confusion than almost anything else in this chapter, so pay close attention to the example below.

arithmetic.py 
x = 10
y = 3

print(x + y)   # 13  — addition
print(x - y)   # 7   — subtraction
print(x * y)   # 30  — multiplication
print(x / y)   # 3.3333...  — division, ALWAYS returns a float
print(x // y)  # 3   — floor division, drops the remainder
print(x % y)   # 1   — modulo, gives ONLY the remainder
print(x ** y)  # 1000 — power (10 to the power of 3)

Click "Run Code" to see the output here.
10 // 3 and 10 % 3, visually 
10 divided by 3  =  3  remainder  1

                    10 // 3       10 % 3
What is... floor division vs. modulo? // keeps the whole-number quotient, always rounding down (10 // 3 is 3). % keeps only the remainder left over (10 % 3 is 1). % is the classic tool for "is this number even?" (n % 2 == 0) and for cycles and wraparounds.

Assignment operators combine an operation with storing the result back into the same variable — you already have a habit of writing i = i + 1; Python gives you a shortcut.

assignment.py 
score = 10
score += 5    # same as: score = score + 5  -> 15
score -= 3    # same as: score = score - 3  -> 12
score *= 2    # same as: score = score * 2  -> 24
print(score)
Go Deeper: no ++ or -- in Python

If you've used C, Java, or JavaScript, you might reach for score++. Python doesn't have that operator at all — it was left out deliberately, partly because ++ in other languages secretly hides two different behaviors (increment-then-use vs. use-then-increment) that cause subtle bugs. In Python, you always write score += 1 instead, which is completely unambiguous.

Quick recap
  • / always returns a float; // floor-divides (rounds down); % returns only the remainder.
  • ** is power: 2 ** 3 is 8.
  • +=, -=, *=, etc. combine an operation with reassignment.
  • Python has no ++ or -- operators.
Sub-topic 2 of 4

2. Relational & logical operators

Relational operators compare two values and always produce a boolean. Logical operators combine multiple boolean results into one final decision.

relational.py 
age = 20
has_ticket = True

print(age >= 18)                    # True
print(age >= 18 and has_ticket)     # True  — BOTH must be true
print(age < 13 or has_ticket)       # True  — only ONE needs to be true
print(not has_ticket)               # False — flips it
What is... a logical operator? and requires every condition to be true for the whole expression to be true. or only needs one condition to be true. not simply flips a boolean: True becomes False and vice versa.
Go Deeper: short-circuit evaluation

Python stops checking as soon as it knows the answer. In age >= 18 and has_ticket, if age >= 18 is already False, Python never even looks at has_ticket — the whole expression is already guaranteed False. This matters in real code: you can safely write x != 0 and 10 / x > 1 because Python won't attempt the division if x is 0.

Quick recap
  • Relational operators (>, <, >=, <=, ==, !=) always return a boolean.
  • and — every condition must be true.
  • or — at least one condition must be true.
  • not — flips a boolean.
Sub-topic 3 of 4

3. Bitwise operators

Every number is stored internally as bits (1s and 0s). Bitwise operators work directly on those bits. This sounds academic, but it's exactly how real permission systems (like Unix file permissions) and low-level flags work — you'll use this for real in this chapter's project.

bitwise.py 
a = 5   # binary: 101
b = 6   # binary: 110

print(a & b)   # 4   — AND: 1 only where BOTH have a 1
print(a | b)   # 7   — OR: 1 where EITHER has a 1
print(a ^ b)   # 3   — XOR: 1 where they DIFFER
print(a << 1)  # 10  — shift bits left (doubles the value)
print(a >> 1)  # 2   — shift bits right (halves it, rounding down)
What is... a bit flag? A bit flag uses individual bits of a single number to represent several independent yes/no switches at once. Instead of four separate boolean variables, you can pack four flags into one number and use & (AND) to check one, and | (OR) to turn one on — this is exactly how Unix file permissions (read=4, write=2, execute=1) work under the hood.
Go Deeper: converting between number systems

Python can show you any integer in binary, octal, or hexadecimal with built-in functions: bin(10) gives '0b1010', oct(10) gives '0o12', and hex(10) gives '0xa'. The prefixes (0b, 0o, 0x) tell Python (and you) which base you're reading, and you can write literals directly in any of these forms in your source code too.

Quick recap
  • & (AND), | (OR), ^ (XOR) operate bit by bit.
  • << and >> shift bits left/right (multiply/divide by powers of 2).
  • bin(), oct(), hex() show a number in another base.
Sub-topic 4 of 4

4. Special operators: is / in

Two more operators come up constantly in real Python code: is checks identity (same object in memory, the concept from Chapter 2), and in checks membership.

special_ops.py 
a = 10
b = 10
print(a is b)          # True  — same small-int object (Chapter 2!)

fruits = ["apple", "banana", "mango"]
print("banana" in fruits)      # True
print("grape" not in fruits)   # True
What is... the difference between == and is? == asks "do these have the same value?" is asks "are these the exact same object in memory?" Two different lists with identical contents are == but not is. This distinction rarely matters for numbers and strings, but matters a lot once you start working with lists and objects in later chapters.
Quick recap
  • is / is not check identity (same object).
  • in / not in check membership (is this value inside this collection).
  • == checks value equality — not the same thing as is.
In the Real World
  • Pagination on every app you use ("Page 3 of 10") is computed with // and % on a total item count and a page size — the exact operators from this chapter.
  • Feature flags at companies like Google and Netflix use bitwise flags exactly like the permissions project — one integer, dozens of independent on/off switches, checked with &.
  • Short-circuit evaluation is a real performance and safety technique — production code guards expensive or risky operations (network calls, division, database lookups) behind a cheap and check first.

Points to Remember

  • / always gives a float; // floor-divides; % gives the remainder; ** is power.
  • Assignment shortcuts (+=, etc.) exist; ++/-- do not.
  • Relational operators return booleans; and/or/not combine them, with short-circuit evaluation.
  • Bitwise operators (&, |, ^, <<, >>) work on individual bits — useful for flags and permissions.
  • is checks identity, in checks membership — neither is the same as ==.
