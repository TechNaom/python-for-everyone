1 Classify I/O-Bound vs. CPU-Bound Easy

Write a function classify_work_type(task_name) that returns "I/O-bound" if task_name is one of "download", "api_call", "database_query", "file_read", or "upload", and "CPU-bound" otherwise.

Constraints: task_name is always a string.
Examples:
classify_work_type("download") → "I/O-bound"
classify_work_type("compute_primes") → "CPU-bound"

Show Solution
solution_1.py 
def classify_work_type(task_name):
    io_tasks = {"download", "api_call", "database_query", "file_read", "upload"}
    if task_name in io_tasks:
        return "I/O-bound"
    return "CPU-bound"

3 Square Every Item Concurrently Medium

Write a function square_all_concurrently(items) that starts one thread per number in items (each thread squaring its number and appending the result to a shared list), joins every thread, and returns the results as a sorted list (sorting removes any dependence on which thread happened to finish first).

Constraints: items is a list of integers, and may be empty.
Examples:
square_all_concurrently([1, 2, 3]) → [1, 4, 9]
challenge_3.py
Show Solution
solution_3.py 
import threading

def square_all_concurrently(items):
    results = []
    threads = [threading.Thread(target=lambda n=n: results.append(n * n)) for n in items]
    for t in threads:
        t.start()
    for t in threads:
        t.join()
    return sorted(results)

4 Safely Total Amounts Across Threads Medium

Write a function safe_total(amounts) that starts one thread per number in amounts, each thread safely adding its number to a shared total using a threading.Lock(), joins every thread, and returns the final total. Must return the mathematically correct sum every time, regardless of thread timing.

Constraints: amounts is a list of integers, and may be empty.
Examples:
safe_total([1, 2, 3, 4]) → 10
safe_total([]) → 0
challenge_4.py
Show Solution
solution_4.py 
import threading

def safe_total(amounts):
    total = 0
    lock = threading.Lock()

    def add(amount):
        nonlocal total
        with lock:
            total += amount

    threads = [threading.Thread(target=add, args=(a,)) for a in amounts]
    for t in threads:
        t.start()
    for t in threads:
        t.join()
    return total

6 Would a Lock Be Needed? Medium

Write a function would_lock_be_needed(threads_read, threads_write) that returns True if a lock would be needed to safely share state given threads_read threads reading and threads_write threads writing the same shared value, and False otherwise. A lock is needed whenever at least one thread writes and more than one thread (reading or writing, combined) touches the value.

Constraints: threads_read and threads_write are non-negative integers.
Examples:
would_lock_be_needed(2, 2) → true (multiple readers and writers)
would_lock_be_needed(3, 0) → false (read-only, nothing being mutated)
would_lock_be_needed(0, 1) → false (only one thread touches it at all)

Show Solution
solution_6.py 
def would_lock_be_needed(threads_read, threads_write):
    total = threads_read + threads_write
    return threads_write >= 1 and total > 1

7 Double Every Item Using a Thread Pool Medium

Write a function double_with_pool(items, max_workers) that uses ThreadPoolExecutor(max_workers=max_workers) as a context manager, maps a doubling function across items with .map(), and returns the result as a plain list (already in input order, since that's what .map() guarantees).

Constraints: items is a list of integers; max_workers is always a positive integer.
Examples:
double_with_pool([1, 2, 3, 4], 2) → [2, 4, 6, 8]
challenge_7.py
Show Solution
solution_7.py 
from concurrent.futures import ThreadPoolExecutor

def double_with_pool(items, max_workers):
    with ThreadPoolExecutor(max_workers=max_workers) as pool:
        return list(pool.map(lambda n: n * 2, items))
