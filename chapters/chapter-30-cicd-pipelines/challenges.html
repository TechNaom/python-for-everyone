1 Does It Trigger on pull_request? Easy

Write a function triggers_on_pull_request(yaml_lines). yaml_lines is a list of strings, each one line of a workflow YAML file. The trigger appears either as a single line like "on: pull_request", or as a multi-line block where "on:" is followed by indented lines like " push:" and " pull_request:". Return True if the workflow triggers on pull_request in either form, else False.

Constraints: yaml_lines is a non-empty list of strings; at most one "on:" section is present.
Examples:
triggers_on_pull_request(["name: CI", "on: pull_request", "jobs:"]) → True
triggers_on_pull_request(["name: CI", "on: push", "jobs:"]) → False
challenge_1.py
Show Solution
solution_1.py 
def triggers_on_pull_request(yaml_lines):
    in_on_block = False
    for line in yaml_lines:
        if line.startswith("on:"):
            rest = line.split(":", 1)[1].strip()
            if rest == "pull_request":
                return True
            in_on_block = True
            continue
        if in_on_block:
            stripped = line.strip()
            if line.startswith((" ", "\t")):
                if stripped in ("pull_request:", "pull_request"):
                    return True
                continue
            else:
                in_on_block = False
    return False

2 Matrix Job Count Easy

Write a function matrix_job_count(dimensions). dimensions is a dict where each key is a matrix dimension name (like "python-version" or "os") and each value is a list of options for that dimension. A matrix strategy runs the job once for every combination across all dimensions. Return the total number of job runs produced.

Constraints: dimensions is a dict with 0 or more keys; every value is a non-empty list.
Examples:
matrix_job_count({"python-version": ["3.9", "3.10", "3.11"]}) → 3
matrix_job_count({"python-version": ["3.9", "3.10"], "os": ["ubuntu-latest", "windows-latest", "macos-latest"]}) → 6
challenge_2.py
Show Solution
solution_2.py 
def matrix_job_count(dimensions):
    total = 1
    for values in dimensions.values():
        total *= len(values)
    return total

3 Extract Job Names Easy

Write a function extract_job_names(yaml_lines) that returns the list of job names defined under the workflow's "jobs:" section, in order. Job names are the first indentation level under jobs: (e.g. " lint:" and " test:" both start a job at 2 spaces of indent); deeper-indented lines like runs-on or step lines are not job names.

Constraints: yaml_lines contains exactly one "jobs:" line; job name lines end with ":" and are indented with spaces.
Examples:
extract_job_names(["jobs:", " lint:", " runs-on: ubuntu-latest", " test:", " runs-on: ubuntu-latest"]) → ["lint", "test"]
challenge_3.py
Show Solution
solution_3.py 
def extract_job_names(yaml_lines):
    names = []
    in_jobs_block = False
    jobs_indent = None
    for line in yaml_lines:
        if line.startswith("jobs:"):
            in_jobs_block = True
            continue
        if in_jobs_block:
            if not line.strip():
                continue
            indent = len(line) - len(line.lstrip(" "))
            if jobs_indent is None and line.startswith(" ") and line.rstrip().endswith(":"):
                jobs_indent = indent
            if jobs_indent is not None and indent == jobs_indent and line.rstrip().endswith(":"):
                names.append(line.strip().rstrip(":"))
            elif jobs_indent is not None and indent < jobs_indent:
                break
    return names

4 Are All Required Checks Passed? Easy

Write a function all_required_checks_passed(required_checks, passed_checks) that returns True only if every check name in required_checks also appears in passed_checks — this is what GitHub actually checks before allowing a PR to merge with branch protection enabled.

Constraints: both arguments are lists of strings; passed_checks may contain extra checks not in required_checks.
Examples:
all_required_checks_passed(["lint", "test"], ["lint", "test", "build"]) → True
all_required_checks_passed(["lint", "test"], ["lint"]) → False
challenge_4.py
Show Solution
solution_4.py 
def all_required_checks_passed(required_checks, passed_checks):
    return all(check in passed_checks for check in required_checks)

7 Is This a Valid Pipeline Stage Order? Medium

Write a function is_valid_stage_order(stages). A well-formed pipeline runs its stages in this canonical order: lint, type_check, test, security_scan, build, deploy_staging, deploy_production (a stage can be skipped, but present stages must appear in this relative order). Given a list of stage names in the order they actually run, return True if that order is consistent with the canonical order, else False.

Constraints: stages is a list of strings, each one of the 7 canonical stage names; no duplicates.
Examples:
is_valid_stage_order(["lint", "test", "deploy_staging", "deploy_production"]) → True
is_valid_stage_order(["deploy_production", "lint", "test"]) → False
challenge_7.py
Show Solution
solution_7.py 
def is_valid_stage_order(stages):
    order = ["lint", "type_check", "test", "security_scan", "build",
             "deploy_staging", "deploy_production"]
    positions = [order.index(s) for s in stages if s in order]
    return positions == sorted(positions)

8 Pick the Right Trigger Medium

Write a function pick_trigger_for_scenario(scenario) that returns the GitHub Actions trigger keyword (as a plain string) matching a described scenario. Support exactly these mappings: "run on every push to any branch" → "push"; "run before merging a pull request" → "pull_request"; "run on a schedule every night" → "schedule"; "let a developer trigger it manually from the Actions tab" → "workflow_dispatch". For any scenario string not in this list, return "unknown scenario".

Constraints: scenario is a string, matched exactly.
Examples:
pick_trigger_for_scenario("run on a schedule every night") → "schedule"
challenge_8.py
Show Solution
solution_8.py 
def pick_trigger_for_scenario(scenario):
    mapping = {
        "run on every push to any branch": "push",
        "run before merging a pull request": "pull_request",
        "run on a schedule every night": "schedule",
        "let a developer trigger it manually from the Actions tab": "workflow_dispatch",
    }
    return mapping.get(scenario, "unknown scenario")
