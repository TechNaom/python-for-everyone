Practice Bank: CI/CD Pipelines

A deeper set of practice problems, organized by topic, for extra reps beyond the main exercises — including scenario-based problems written in the same style you'll meet in real interviews. This chapter teaches GitHub Actions (YAML-based CI/CD), not Python syntax, so every problem here is a Python function that parses, validates, or reasons about a CI/CD concept using plain data standing in for a workflow file or pipeline state. Standard library only — no installs needed.

Why reason about YAML instead of running real workflows? This site runs your code in a browser sandbox with no GitHub Actions runner and no yaml package available, so every problem models a workflow as plain Python data (a list of strings standing in for YAML lines, dicts for matrix dimensions, booleans for pass/fail state) instead of literally executing a pipeline. Every behavior these problems describe was checked against real GitHub Actions semantics first — the mental model is genuinely correct, just expressed in Python so it can be auto-graded. To practice for real, create a repo, add a workflow file under .github/workflows/, and watch it run in the Actions tab.
