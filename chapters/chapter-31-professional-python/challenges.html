1 Log Level Number to Name Easy

Constraints: level_number is one of 10, 20, 30, 40, 50.
Examples:
level_name_for_number(30) → "WARNING"
level_name_for_number(10) → "DEBUG"

def level_name_for_number(level_number):

2 Would This Message Be Emitted? Easy

Return True if the message would be emitted, else False.

Constraints: both arguments are ints among 10, 20, 30, 40, 50.
Examples:
would_message_be_emitted(30, 20) → False
would_message_be_emitted(30, 40) → True

Show Solution
solution_2.py 
def would_message_be_emitted(logger_level, message_level):
    return message_level >= logger_level

3 Parse Real CLI Arguments Medium

Write a function parse_cli_args(argv) that builds an argparse.ArgumentParser with one required positional argument "filename", one optional flag "--verbose" (action="store_true"), and one optional flag "--output" that takes a value (default None). Parse the list argv with it and return a dict with keys "filename", "verbose", and "output" holding the parsed values.

Constraints: argv is a list of strings that always includes exactly one filename and zero or more of the optional flags.
Examples:
parse_cli_args(["data.csv"]) → {"filename": "data.csv", "verbose": False, "output": None}
parse_cli_args(["data.csv", "--verbose"]) → {"filename": "data.csv", "verbose": True, "output": None}
challenge_3.py
Show Solution
solution_3.py 
import argparse

def parse_cli_args(argv):
    parser = argparse.ArgumentParser()
    parser.add_argument("filename")
    parser.add_argument("--verbose", action="store_true")
    parser.add_argument("--output", default=None)
    ns = parser.parse_args(argv)
    return {"filename": ns.filename, "verbose": ns.verbose, "output": ns.output}

4 A Count-Style Verbosity Flag Medium

Write a function count_verbosity(argv) that builds an argparse.ArgumentParser with one optional flag "--verbose" using action="count" and default=0, so passing the flag multiple times increases a count. Parse argv with it and return the resulting count as an int.

Constraints: argv is a list of strings containing zero or more "--verbose" entries and nothing else.
Examples:
count_verbosity([]) → 0
count_verbosity(["--verbose", "--verbose"]) → 2
challenge_4.py
Show Solution
solution_4.py 
import argparse

def count_verbosity(argv):
    parser = argparse.ArgumentParser()
    parser.add_argument("--verbose", action="count", default=0)
    ns = parser.parse_args(argv)
    return ns.verbose

5 Count Emitted Messages Medium

Write a function count_emitted_messages(logger_level, message_levels). Given a logger's numeric threshold level and a list of numeric levels for messages that code attempted to log, return how many of those messages would actually be emitted (i.e. how many have a level greater than or equal to logger_level).

Constraints: logger_level is an int; message_levels is a list of ints (may be empty).
Examples:
count_emitted_messages(30, [10, 20, 30, 40, 50]) → 3
challenge_5.py
Show Solution
solution_5.py 
def count_emitted_messages(logger_level, message_levels):
    return sum(1 for lvl in message_levels if lvl >= logger_level)

6 Classify a Dependency Pin Style Easy

Write a function dependency_pin_style(version_spec) that classifies a dependency version specifier string's pin style. Return "exact" if it contains "==", "compatible" if it contains "~=", "minimum" if it contains ">=" (and neither of the above), or "unpinned" if it's an empty string or matches none of these (no version constraint at all, just the bare package name).

Constraints: version_spec is a string; check in this priority order: exact, then compatible, then minimum, else unpinned.
Examples:
dependency_pin_style("==2.34.2") → "exact"
dependency_pin_style("") → "unpinned"
challenge_6.py
Show Solution
solution_6.py 
def dependency_pin_style(version_spec):
    if version_spec == "":
        return "unpinned"
    if "==" in version_spec:
        return "exact"
    if "~=" in version_spec:
        return "compatible"
    if ">=" in version_spec:
        return "minimum"
    return "unpinned"

7 Parse pip freeze Output Medium

Write a function parse_pip_freeze(freeze_output) that takes the multi-line string output of pip freeze (one "package==version" per line) and returns a dict mapping each package name to its version string. Ignore blank lines.

Constraints: freeze_output is a string; each non-blank line has exactly one "==".
Examples:
parse_pip_freeze("requests==2.34.2\nidna==3.18") → {"requests": "2.34.2", "idna": "3.18"}
challenge_7.py
Show Solution
solution_7.py 
def parse_pip_freeze(freeze_output):
    result = {}
    for line in freeze_output.splitlines():
        line = line.strip()
        if not line:
            continue
        name, version = line.split("==")
        result[name] = version
    return result
